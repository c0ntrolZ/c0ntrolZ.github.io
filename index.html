Projects & Research

01 AI Student Assistant Chatbot Mar – Jun 2025

Fine-tuned a large language model to enhance reasoning, enabling it to function as an AI student assistant. Developed and compared four approaches: DPO, MCQA, RAG, and Quantization.

Python PyTorch Transformers LLM Fine-Tuning DPO RAG
Read Paper

Explored four distinct approaches to aligning a general-purpose LLM for STEM tutoring. DPO (Direct Preference Optimization) was used for preference-based alignment; MCQA fine-tuning targeted structured question-answering over course material; RAG grounded answers in lecture notes and slides; and quantization reduced inference cost while preserving accuracy. Models were evaluated on a held-out set of student questions across multiple EPFL courses.

02 Privacy-Preserving Marketplace on Blockchain Sep – Dec 2025

Designed and formalized a complete cryptographic protocol for an anonymous-review marketplace on Ethereum. Implemented via Solidity smart contracts using zk-SNARKs (Groth16 circuits) and Merkle-tree constructions.

Solidity zk-SNARKs Circom JavaScript Python
View Code Read Paper

Buyers can leave anonymous reviews without revealing their identity to the seller or any observer. The core mechanism uses Groth16 zk-SNARK circuits compiled with Circom to prove purchase eligibility without disclosing the buyer's address. Merkle trees track eligible reviewers on-chain. Solidity contracts handle the full purchase-review lifecycle on Ethereum's Sepolia testnet, with a FastAPI backend and Node.js workers managing proof generation.

03 Machine Learning for FPGA Routing Oct – Dec 2024

Research on applying ML models to assist the FPGA routing process. Collected a domain-specific dataset, explored new data features and task formulations to improve prediction accuracy.

Python ML Research
Read Paper

Investigated whether ML models could predict FPGA routing cost metrics (wirelength, congestion) from netlists, enabling faster hardware design iteration. A custom dataset was collected from real FPGA runs. Gradient boosting and neural network models were compared across different feature sets — including graph-based netlist features — and regression targets. Feature engineering showed significant improvements over naive baselines.

04 Secure Distributed Computing Oct – Dec 2024

Built a distributed machine learning system (training and inference) in Go over a peer-to-peer network, using homomorphic encryption to preserve data privacy during computation.

Go Distributed Systems Homomorphic Encryption ML

Trains a shared ML model across multiple nodes without any node revealing its private dataset. Nodes exchange encrypted model updates rather than raw data over a P2P network built in Go. The BFV homomorphic encryption scheme ensures updates remain encrypted during aggregation, so no observer can infer individual training samples. Validated on a linear regression task across 4 simulated nodes.
